# ChatGPT大战期末周 - 完整项目报告

## 一、项目介绍

### 1.1 项目背景
本项目是针对现代大学生期末复习困境而设计的一款网页策略模拟游戏。游戏以轻松幽默的方式还原了大学生在14天期末复习中面临的各种抉择和挑战。

### 1.2 项目名称
**ChatGPT大战期末周** - 一个融合学习、娱乐、时间管理于一体的交互式游戏

### 1.3 项目类型
- **游戏类型**: 文字策略模拟 + 生存经营 + 随机事件
- **开发平台**: Web网页端（HTML5 + CSS3 + JavaScript）
- **后端技术**: Node.js + HTTP服务器
- **数据存储**: JSON文件数据库

### 1.4 创意亮点
✨ **高度还原真实场景** - 完整模拟大学生期末生活的各个方面
✨ **策略性决策机制** - 在学习、娱乐、休息之间权衡，考验玩家的资源管理能力
✨ **动态随机事件系统** - 7种随机事件增加游戏变数和重玩价值
✨ **渐进式解锁系统** - AI技能树5个等级的进阶体验
✨ **多层级结局评价** - 5种不同结局，提供不同的游戏体验和反思空间

### 5.2 行动执行逻辑

```javascript
function performAction(actionId) {
  // 防护检查
  if (gameState.gameOver || !gameState.gameStarted) return;

  // 获取行动配置
  const action = actions.find(a => a.id === actionId);
  if (!action) return;

  // 应用行动效果
  const effect = action.effect;
  gameState.gpa = Math.max(0, Math.min(100, gameState.gpa + effect.gpa));
  gameState.stamina = Math.max(0, Math.min(100, gameState.stamina + effect.stamina));
  gameState.mood = Math.max(0, Math.min(100, gameState.mood + effect.mood));
  gameState.aiSkill = Math.max(0, Math.min(100, gameState.aiSkill + effect.ai));

  // 分配课程进度（所有课程均匀增长）
  if (effect.gpa > 0) {
    const courses = Object.keys(gameState.courses);
    courses.forEach(course => {
      gameState.courses[course] += Math.floor(effect.gpa / 2);
    });
  }

  // 记录日志
  addLog(`执行了"${action.name}"行动`, 'action');

  // 70%概率触发随机事件
  if (Math.random() < 0.7) {
    setTimeout(() => triggerRandomEvent(), 500);
  } else {
    advanceDay();
  }

  updateUI();
}
```

**核心特性**：
- **数值钳制**：`Math.max(0, Math.min(100, ...))`保证属性不超界
- **随机性**：70%事件触发概率提高游戏变数
- **延时处理**：`setTimeout`使事件显示更自然

### 5.5 后端登录接口

```javascript
async function handleApi(req, res) {
  // 登录接口
  if (req.method === "POST" && req.url === "/api/login") {
    try {
      const { username, password } = await readBody(req);
      const cleanName = String(username || "").trim();
      const cleanPassword = String(password || "");

      // 验证输入
      if (!/^[\w\u4e00-\u9fa5]{2,16}$/.test(cleanName)) {
        return sendJson(res, 400, { message: "用户名需为 2-16 位中文、英文、数字或下划线" });
      }

      if (cleanPassword.length < 4) {
        return sendJson(res, 400, { message: "密码至少 4 位" });
      }

      const db = loadDb();
      let user = db.users.find((item) => item.username === cleanName);

      // 新用户自动注册
      if (!user) {
        user = {
          username: cleanName,
          passwordHash: hashPassword(cleanPassword),
          bestScore: 0,
          createdAt: new Date().toISOString()
        };
        db.users.push(user);
        saveDb(db);
      } 
      // 老用户验证密码
      else if (!verifyPassword(cleanPassword, user.passwordHash)) {
        return sendJson(res, 401, { message: "密码错误" });
      }

      // 生成会话令牌
      const token = crypto.randomUUID();
      sessions.set(token, user.username);
      
      return sendJson(res, 200, { token, user: publicUser(user) });
    } catch (error) {
      return sendJson(res, 400, { message: error.message });
    }
  }
}

// 密码加密函数
function hashPassword(password, salt = crypto.randomBytes(12).toString("hex")) {
  const hash = crypto.pbkdf2Sync(password, salt, 100000, 32, "sha256").toString("hex");
  return `${salt}:${hash}`;
}
```

**安全特性**：
- PBKDF2加密（工业级标准）
- 随机Salt防止彩虹表攻击
- 输入验证（格式检查）
- UUID令牌（无法预测）

**遗憾之处**：
- 原计划的"不同难度模式"没有实现
- 游戏音效和背景音乐缺失（主要是时间限制）
- 没有实现排行榜的实时更新和社交分享功能

**对同学的建议**：
- 游戏开发不是追求复杂，而是追求深度
- 每一个系统都应该有明确的目的和用户价值
- 测试和迭代比快速开发更重要

### 组员3：后端工程师 - [请补充名字]

**服务器开发日记**：

选择用Node.js的原生HTTP模块而不是Express框架，这个决定看似简洁，实际上在实现过程中发现了很多细节：

**后端架构心得**：

1. **数据安全**
   - PBKDF2加密算法的参数调优（100,000次迭代）
   - 随机Salt的生成确保相同密码不会产生相同哈希
   - 令牌(Token)基于UUID的会话管理

2. **API设计**
   ```
   POST /api/login - 用户认证
   GET /api/me - 获取当前用户信息
   POST /api/score - 保存游戏分数
   POST /api/game/save - 保存游戏进度
   GET /api/game/load - 加载游戏进度
   ```

3. **性能优化**
   - JSON序列化/反序列化的效率
   - 内存中缓存sessions避免重复读取
   - 大请求体的大小限制（1MB）

**技术选择的权衡**：
- JSON数据库 vs 真实数据库：选择JSON是为了快速原型开发，生产环境应该用MongoDB或PostgreSQL
- 同步文件操作 vs 异步：当前实现是同步的，高并发时应该异步化

**踩坑记录**：
1. 第一次没有URL解码，导致中文用户名失败
2. 密码验证时忘记处理盐值拆分，导致总是验证失败
3. CORS问题（跨域请求）花了2小时才解决

**对自己的评价**：
- 代码模块化做得还不够好
- 错误处理和日志记录不够完善
- 应该写更多的单元测试

### 组员4：游戏设计师 - [请补充名字]

**游戏设计文档之外的思考**：

看似轻松的"压力模拟游戏"，实际上承载了对当代大学生困境的深刻反思。

**设计理念**：

1. **真实性**
   - 四门课程是真实的专业课程
   - 四种属性反映了学生的真实状态
   - 随机事件都来自真实经历

2. **平衡设计**
   - 不是简单地鼓励学生"只要学习"
   - 承认娱乐和休息的必要性
   - "睡觉"行动的GPA不变，体现了休息的价值

3. **心理学应用**
   - 倒计时带来的压迫感
   - 进度条的成就感
   - 随机事件的惊喜感
   - 多结局的反思空间

**未实现的创意**：
- 每位玩家的AI会"学习"你的策略并变得更聪明
- 多人合作模式（共同对抗期末周）
- 隐藏结局：
  - "通宵修仙"：在最后一天突然突击学习
  - "提前摆烂"：故意在第一周放弃
  - "ChatGPT信徒"：AI熟练度100%但GPA低

**对游戏工业的思考**：
- 游戏不一定要3A级别的画面才有价值
- 文字游戏通过想象力可以打动人心
- "即时反馈"是游戏成瘾性的关键

**最大的收获**：
理解了为什么《Flappy Bird》这样简单的游戏能成为现象级产品，因为设计的精妙度不在于复杂度，而在于约束与自由的完美平衡。
